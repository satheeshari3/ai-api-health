# Coding Standards for AI API Health Monitor\
\
This document ensures **consistent, maintainable, and readable code** across the project.\
\
## Python (Backend)\
- Follow PEP8 conventions\
- Use type hints wherever possible\
- Include docstrings for all functions and classes\
- Structure Flask routes logically\
- Validate API inputs rigorously\
\
## JavaScript / React (Frontend)\
- Use functional components + hooks\
- Keep components small and reusable\
- Follow consistent folder structure (components/, pages/, utils/)\
- Use clear variable naming and avoid abbreviations\
- Document complex components with comments\
\
## Logging & AI\
- Use structured logging (JSON format recommended)\
- Ensure AI prompts are modular and reusable\
- Keep AI instructions in separate `.md` files\
- Never hardcode sensitive API keys or tokens}
